# DopaList 需求文档

## 1. 应用概述

### 1.1 应用名称

DopaList

### 1.2 应用描述

一款极简风格的待办事项管理应用，融合多巴胺奖励机制，帮助用户高效完成任务的同时获得意外惊喜。\n
## 2. 视觉风格

### 2.1 设计语言

参考线性设计风格，极致简约，黑白灰主色调（锌灰色系），强调细眉和微阴影效果。\n
### 2.2 排版

字体清晰，信息密度高，营造专业高效的视觉体验。\n
### 2.3 组件库

深度使用 Shadcn UI 组件，包括：Card、Button、Checkbox、Badge、Dialog、Toast 等。\n
## 3. 核心功能

### 3.1 任务管理

#### 3.1.1 任务添加

- 支持输入框快速添加任务
- 按回车键提交新任务

#### 3.1.2 任务属性

- 标题：必填项
- 截止日期：支持选择任务截止日期
- 优先级：支持 P1（红色标识）、P2（黄色标识）、P3（无标识）三级优先级
- 类别：所有任务必须归属于用户自定义的类别
- 初始化日期：自动记录任务创建日期

#### 3.1.3 任务操作

- 勾选完成任务
- 删除任务

#### 3.1.4 类别管理

- 支持用户自定义创建类别
- 支持编辑类别名称
- 支持删除类别（关联任务需处理）\n- 支持在类别下查看和管理任务

### 3.2 多巴胺奖励机制（核心亮点）\n
#### 3.2.1 奖励触发逻辑

当用户勾选完成任务时，触发随机概率判定（基于 Math.random）：\n
- **30% 概率：小奖励**\n  - 弹出 Shadcn Toast 或 Dialog，展示文案（如“休息5分钟”、“喝杯水”）\n  - 播放成功音效反馈
- **10% 概率：超级大奖**\n  - 屏幕中央弹出醒目的动画弹窗，展示文案（如“看一集剧！”、“今晚不加班！”）\n  - 触发全屏五彩纸屑效果
- **60% 概率：正常完成**\n  - 简单的勾选动画反馈

#### 3.2.2 奖励内容来源

奖励文案从预设数据库中随机抽取，奖励库用户自己编辑。\n
### 3.3 数据持久化

所有数据保存在浏览器 LocalStorage 中，刷新页面不丢失。\n
## 4. 交互细节

### 4.1 动画效果

- 使用 Framer Motion 实现任务列表的平滑出现和删除动画
- 完成任务时提供爽快的交互反馈

### 4.2 音效反馈

- 完成任务时播放相应的音效
- 奖励触发时播放音效增强体验

## 5. 响应式设计

### 5.1 PC 端

- 截止日期和优先级设置放在同一行
- 保持原有其他设计不变

### 5.2 移动端

- 截止日期和优先级保持原有布局不变
- 优化任务列表在移动端的展示效果
- 增加类别管理功能，支持用户自定义类别
- 适配移动端操作习惯，提供友好的交互体验
